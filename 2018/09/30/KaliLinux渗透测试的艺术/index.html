Kali Linux渗透测试的艺术
发布于 2018-09-30 / 更新于 2019-01-08 / Kali Linux / 热度 0 / 喜欢 0

Kali Linux发展

  1. Kali Linux = Back Track + IWHAX + WHOPPIX + Auditor
  2. 2013年3月12日 Kali Linux 1.0 问世

Kali Linux 特色

  1. 基于Debian的Linux发行版
  2. 集成300多个渗透测试程序
  3. 支持绝大多数的无线网卡
  4. 修改了内核以支持无线数据包注入
  5. 所有的软件包都有研发团队的PGP签名
  6. 用户可以自制满足各自需求的Kali Linux发行版
  7. 支持基于ARM的硬件系统

Kali Linux工具包

工具程序 功能
信息收集 收集目标的DNS、IDS/IPS、网络扫描、操作系统、路由、SSL、SMB、VPN、VoIP、SNMP信息和E-mail地址
漏洞评估 扫描目标系统漏洞、部分工具可以检测Cisco网络系统缺陷、有的可以评估各种数据库系统安全问题
Web应用 包括CMS（内容管理系统）扫描器、数据库漏洞利用工具、Web应用模糊测试、Web应用代理、Web爬虫及Web漏洞扫描器
密码攻击 在线攻击、离线破解
漏洞利用 攻击网络、Web和数据库漏洞的软件，都属于漏洞利用（exploitation）工具、社会工程学攻击
网络监听 监听网络和Web流量，网络监听需要进行网络欺骗，所以Ettercap和Yersinia这类软件也算
访问维护 帮助渗透人员维持他们对目标主机的访问权，某些情况下，渗透人员必须先获取主机的最高权限才能安全这类软件。这类软件包括用于Web应用和操作系统安装后门的程序，以及隧道类工具
报告工具 撰写渗透测试的报告文件
系统服务 渗透测试时可能用到的常用服务类软件，包括Apache服务、MySQL服务、SSH服务和Metasploit服务
无限攻击 可攻击蓝牙、RFID/NFC和其他无线设备
逆向工程 调试程序或反汇编
压力测试 测试网络、无线、Web和VoIP系统的负载能力
硬件破解 调试Android和Arduino程序
法证调查 电子取证。可用于制作硬盘磁盘镜像、文件分析、硬盘镜像分析。如需使用这类程序，首先要在启动菜单里选择Kali Linux Forensics->No Drives or Swap Mount。在开启这个选项后，Kali Linux不会自动加载硬盘驱动器，以保护硬盘数据的完整性

为了降低渗透测试人员筛选工具的难度，Kali Linux单独划分了一类软件–Top 10 Security Tools，即10大首选安全工具。分别是aircrack-ng、burp-suite、
hydra、john、maltego、metasploit、nmap、sqlmap、wireshark和zaproxy

脆弱系统（vulnerable server）

一台存在很多漏洞的虚拟主机（操作系统、网络、Web应用服务），在虚拟机里安装metasploitable2，研发团队是Rapid7旗下著名的HD Moore

渗透测试(penetration testing,pentest)

渗透测试类型 说明
黑盒测试 安全审计员在不清楚被测单位的内部技术构造的情况下，从外部评估网络基础设施的安全性
白盒测试 审计员可以获取被测单位的各种内部资料甚至不公开资料，视野更为广阔。时间、成本以及发现、解决安全弱点的技术门槛全面低于黑盒测试

脆弱性评估与渗透测试

1.脆弱性评估（漏洞扫描）：通过分析企业资产面临威胁的情况和程度，评估内部和外部的安全控制的安全性。这种技术上的信息系统评估，不仅要揭露现有防范措施里存在的风险，而且要提出多重备选的补救策略，并将这些策略进行比较。内部的脆弱性评估可以保证内部系统的安全性，而外部的脆弱性评估则用于验证边界防护的有效性。一体化的安全弱点管控平台（vulnerability management）平台带有可自动更新的漏洞数据库，能够测试不同类型的网络设备，而且不会影响配置管理和变更管理的完整性。

2.渗透测试不仅要识别目标的弱点，还涉及在目标系统上进行漏洞利用、权限提升、访问维护。脆弱性评估虽然可以充分发现目标系统里的缺陷，但是不会考虑衡量这些缺陷对系统造成的危害。另外，相比脆弱性评估，渗透测试更倾向于入侵，会可以使用各种技术手段利用安全漏洞；所以渗透测试可能对生产环境带来实际的破坏性影响。而脆弱性评估则是以非入侵的方式，定性、定量地识别已知安全缺点。

区别
渗透测试 1. 不仅要识别目标的弱点，还涉及在目标系统上进行漏洞利用、权限提升、访问维护 2. 渗透测试更倾向于入侵，会可以使用各种技术手段利用安全漏洞 3.可能对生产环境带来实际的破坏性影响
脆弱性评估 1.可以充分发现目标系统里的缺陷 2.以非入侵的方式，定性、定量地识别已知安全缺点

安全评估方法论

安全评估方法论 说明 种类 特性及优势
开源安全测试方法论OSSTMM 国际公认的安全测试和安全分析标准，把安全评估工作划为4组：范围（scope）、信道（channel）、索引（index）和矢量（vector） 盲测、双盲测、灰盒测试、双灰盒测试、串联测试、逆向测试
信息系统安全评估框架
开放式Web应用安全项目
Web应用安全联合威胁分类（WASC-TC）
渗透测试执行标准(PTES)
  1. OWASP(Open Web Application Security Project)推出的测试框架主要面向应用安全的安全评估。
  2. PTES（Penetration Testing Execution Standard）能够指导所有类型的渗透测试工作

测试步骤

  1. 范围界定
  2. 信息收集
  3. 目标识别
  4. 服务枚举
  5. 漏洞映射
  6. 社会工程学
  7. 漏洞利用
  8. 提升权限
  9. 访问维护
  10. 文档报告
