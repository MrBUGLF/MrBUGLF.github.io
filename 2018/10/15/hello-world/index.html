Hexo + Github搭建博客
发布于 2018-10-15 / 更新于 2019-01-08 / Hexo / 热度 0 / 喜欢 0

搭建环境配置

  1. 安装node

    $node -v
v10.8.0
$npm -v
6.2.0

  2. 安装git

    $git --version
git version 2.8.0.windows.1

  3. 使用npm安装Hexo

    npm install -g hexo-cli

  4. 搭建本地Hexo博客

    //在一个文件夹下初始化博客
>hexo i blog
//切换到站点根目录
>cd blog
//生成博客generator
>hexo g
//启动本地Hexo server
>hexo s

    之后在浏览器中输入localhost:4000查看效果

    可以更换不同的主题，选择你喜欢的，找到github地址

    然后在站点根目录下输入

    git clone https://github.com/iissnan/hexo-theme-next themes/next

    完成后，打开站点配置文件_config.yml，找到theme字段，替换值为next，可选择模式Scheme

    • Muse ：默认Scheme，黑白主调
    • Mist：Muse 的紧凑版本，整洁有序的单栏外观
    • Pisces：双栏Scheme

    在blog站点根目录下输入

    // 1.清除缓存

    hexo clean

    //2.重新生成代码

    hexo g

    //3.部署到本地

    hexo s

    再打开浏览器访问

    更多设置前往官网

  5. 部署到GitHub上

    • 注册一个GitHub账号，假设用户名为your_name，然后创建一个新repository，项目名为your_name.github.io`

    • 可以自己创建page如果是第一次创建的话，可以采用下面的选择

    • 生成SSH密钥，

      1. cd ~/.ssh //查看.ssh文件夹是否存在

      2.ls //查看该文件夹下内容

      3.ssh-keygen -t rsa -C “XXXX@qq.com，连续回车，生成密钥，有两个文件id_rsa和id_rsa.pub

      4.eval "$(ssh-agent -s)"添加到ssh-agent

      5.ssh-add ~/.ssh/id_rsa添加SSH key到ssh-agent

      6.登录Github，点击头像下的seetings，添加ssh，把id_rsa.pub文件内容复制进去

      7.ssh -T git@github.com查看是否添加成功

      8.如果ssh配置失败，首先清除所有的key-pair，输入ssh-add -D，rm -r ~/.ssh ，再删除GitHub上的public-key，之后再重新生产密钥对

    • 修改_config.yml配置文件，添加以下内容（根据自己的情况而定），与GitHub同步

      deploy:
  type: git
  repository: git@github.com:MrBUGLF/MrBUGLF.github.io.git
  branch: master

      再安装一个插件，部署到GitHub

      npm install hexo-deployer-git --save
hexo d //部署

      稍等一下就可以通过your_name.github.io访问了

  6. 绑定域名

    • 首先得申请一个云服务器和域名，然后在域名解析那里添加一条记录

      然后在本地博客根目录下source/路径下新建一个CNAME文件，在里面写上配置的路径，如blog.improvecfan.cn，之后再重新发布就可以了
